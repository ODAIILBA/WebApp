## 🔄 How Demo Pages Work

### Technical Implementation

1. **Route Handler**: `/admin/*` catch-all route in `src/index.tsx`
2. **Config Check**: Looks up `adminPageConfigs[path]`
3. **Fallback**: If no config found, returns `AdminPlaceholder` component
4. **Dynamic**: Automatically extracts category and feature from URL path

### Component Structure

```typescript
AdminPlaceholder(routePath, pageTitle)
├── Sidebar (AdminSidebarAdvanced)
├── Breadcrumb navigation
├── Icon circle (animated)
├── Category badge
├── Page title
├── Status pill ("In Entwicklung")
├── Feature cards (3)
│   ├── Geplante Features
│   ├── Enterprise-Ready
│   └── Performance
└── Action buttons
    ├── Back to Dashboard
    └── View All Features
```

## 🚀 Next Steps to Make Pages Live

To convert a demo page to a live, functional page:

1. **Create database migration** (if needed)
   ```sql
   -- migrations/XXXX_feature_name.sql
   CREATE TABLE feature_name (...);
   INSERT INTO feature_name VALUES (...);
   ```

2. **Add API endpoints** in `src/index.tsx`
   ```typescript
   app.get('/api/feature', async (c) => {
     const data = await c.env.DB.prepare('SELECT * FROM feature').all();
     return c.json(data);
   });
   ```

3. **Create admin component** in `src/components/`
   ```typescript
   export function AdminFeatureName() {
     return `<!DOCTYPE html>...`;
   }
   ```

4. **Add route handler** in `src/index.tsx`
   ```typescript
   app.get('/admin/feature', async (c) => {
     const data = await fetchData(c.env.DB);
     return c.html(AdminFeatureName(data));
   });
   ```

5. **Rebuild and test**
   ```bash
   npm run build
   pm2 restart webapp
   ```
